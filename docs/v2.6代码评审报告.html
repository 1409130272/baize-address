📋改动背景

DeepSeek V4 正式版（7 月中旬上线）引入峰谷定价机制：

  • 高峰 9:00–12:00、14:00–18:00（北京时间），单价约为平时 2 倍
  • 平时 其余时段按原单价

4.4 新增 _is_peak_now — 北京时间判定

def _is_peak_now(self): """判断当前北京时间是否处于高峰时段。未配置时返回 False""" if not self.peak_ranges: return False from datetime import datetime, timezone, timedelta tz = timezone(timedelta(hours=8)) # 北京时间 UTC+8 now = datetime.now(tz) minute_of_day = now.hour * 60 + now.minute for start_min, end_min in self.peak_ranges: if start_min <= minute_of_day < end_min: return True return False

🔍评审意见

✅ 向后兼容性设计良好 兼容
所有新参数都带默认值：peak_hours 默认空字符串 → peak_ranges=[] → 永远走平时单价；price_peak_* 默认 None → 回退为平时单价。普通模型（如 deepseek-chat）只需留空 peak_hours 即可，零破坏性。
✅ 时区处理正确 逻辑
使用 timezone(timedelta(hours=8)) 显式构造北京时间，不依赖服务器本地时区。即使部署在 UTC 时区的容器中也能正确判断高峰时段。
✅ 输入解析鲁棒 逻辑
_parse_peak_hours 对非法格式（abc、25:00-26:00）、跨日区间（23:00-06:00）均跳过，不会抛异常；空字符串返回 [] 等价于不开峰谷。容错性好。
✅ 字段一致性 兼容
_build_cost_info、_zero_cost、ResolveResult 默认结构、AICostInfo Pydantic 模型四处都同步增加了 isPeak 字段，未调用 AI 时也返回 isPeak=false，响应结构稳定。
⚠️ 配置留空陷阱 配置
configparser.getfloat 对字段值为空（如 price_peak_output =）会抛 ValueError。建议"不开峰谷"时直接从 ini 删除高峰三字段（走 fallback），或填与平时相同的值，避免留空值。文档已说明此点。
⚠️ 时区导入位置 逻辑
datetime 的 import 写在 _is_peak_now 方法内部，每次调用都会执行 import 语句（虽然 Python 有 module cache，开销极小）。这是为了规避循环依赖的保守写法，可接受；如追求整洁也可提到模块顶部。
✅ 单价配置可观察 逻辑
响应新增 isPeak 字段，下游可清晰识别本次计费所处时段，便于成本分析、对账与告警，符合 v2.5 "费用透明"的演进方向。
✅ 改动范围克制 兼容
未引入新依赖、未重构既有结构、未修改 AI 调用流程。仅在费用计算环节插入时段判定，改动面最小化，回归风险低。
